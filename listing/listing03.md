Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
<nil>
true

Код определяет функцию Foo, которая возвращает
*os.PathError, инициализирующийся со значением nil.

Функция «main» вызывает «Foo» и присваивает возвращаемое значение переменной «err».
Затем значение `err` печатается и проверяется, равно ли оно nil.

type iface struct {
	tab  *itab
	data unsafe.Pointer
}

Экземпляр интерфейса  представляет  собой структуру
из двух указателей  - указатель на тот объект, который фактически скрывается за интерфейсом,
и указатель на таблицу, помогающую транслировать обращение к методам, объявленным в интерфейсе, 
к фактическим методам, объявленным в объекте, от которого образован данный интерфейс. 
У пустого интерфейса же присутствует только указатель на объект, без таблицы - 
невидимый плащ над конкретным типом, который прячет конкретику
и не даёт никакого понимания о поведении
```
